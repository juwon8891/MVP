<h4>Commit convention rule</h4>

> 1. feat : 새로운 기능 ✨
> 2. fix : 버그 수정 🐛
> 3. build : 빌드 관련 파일 수정 🏗️
> 4. chore : 그 외 자잘한 수정 ✏️
> 5. ci : CI관련 설정 수정 👷
> 6. docs : 문서 수정 📝
> 7. style : 코드 스타일 혹은 포맷 등 💄
> 8. refactor :  코드 리팩토링 ♻️
> 9. test : 테스트 코드 수정 ✅
